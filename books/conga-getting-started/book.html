Every great API integration begins with a single login…

Welcome, brave developer, to the enchanted realm of the Conga Advantage Platform. Today we embark on a noble quest: to create an Integration User and obtain the legendary Client ID and Client Secret — the twin keys that unlock the gates of API access.

Conga recommends the Integration User approach over API Connections for most use cases. Fear not! This guide shall be your faithful map.

Chapter 3
3 Add a New Integration User
Admin Console → Users → Add

Click the Add button. The User Details form appears — defaulting to the “Internal” user type. We need to change that.

The User Details form — defaults to “Internal” user type

Click the User Type dropdown and select Integration. The form simplifies — Integration users don’t need a Username or External ID.

The User Type dropdown — select “Integration”
What is an Integration User?

An Integration user is a non-human system account designed for API access, data syncing, and third-party integrations. It authenticates via Client Credentials — no browser login required.

Chapter 4
4 Save & Capture Your Client ID & Secret

Click Save. A success toast appears: “The data is saved successfully” followed by “Credentials generated successfully.” A Credentials popup immediately reveals the two sacred tokens:

Client ID

Your application’s public identity. Safe to store in config files and environment variables.

Client Secret

Your application’s private password. Guard this with your life!

The Credentials popup — copy both values immediately!

Critical Warning!

The Client Secret is shown only once. Once you dismiss this popup, the secret cannot be recovered. Copy it immediately and store it in a secrets manager or secure environment variable. If lost, you can click Generate Secret on the user’s detail page to regenerate credentials.

Write it down! No, not on a sticky note. Somewhere actually secure.

Chapter 5
5 Make Your First Authenticated API Call

With your Client ID and Secret in hand, you can now authenticate via the OAuth 2.0 Client Credentials flow.

Step 5a — Request an Access Token

# Exchange credentials for an access token curl -X POST https://login-rls.congacloud.com/api/v1/auth/connect/token \ -H "Content-Type: application/x-www-form-urlencoded" \ -d "client_id=$CLIENT_ID" \ -d "client_secret=$CLIENT_SECRET" \ -d "grant_type=client_credentials" \ -d "scope=sign"

The scope parameter depends on which Conga API you’re accessing:

ScopeAPI
signConga Sign
doc-gen.composerConga Composer
clmConga CLM (Contract Lifecycle)

Step 5b — Response

# You'll receive: { "access_token": "eyJhbGciOiJSUzI1NiIs...", "token_type": "Bearer", "expires_in": 3600, "scope": "sign" }

Step 5c — Call the API

# Use the Bearer token in subsequent requests curl https://api.congacloud.com/v1/your-endpoint \ -H "Authorization: Bearer $ACCESS_TOKEN" \ -H "Content-Type: application/json"
Token Tip

Tokens expire after 3600 seconds (1 hour). Your app should track expires_in and request a new token before the old one lapses. Don’t request a new token on every API call — reuse it!

Bonus Chapter
! Regenerating a Lost Client Secret

Lost your Client Secret? It happens to the best of us. Here’s how to get a new one:

1. Navigate to Admin Console → Users

2. Find your Integration User in the list

3. Click the External Identifier hyperlink for that user

4. Click Save & Regenerate Client Secret

5. Copy the new Client ID and Client Secret from the popup

Important!

Regenerating a secret invalidates the old one immediately. Update the new secret in all connected applications, or they will lose API access.

+ Alternative: Conga API Connections

There’s a second way to obtain API credentials, via the API Connections tab:

Admin Console → Organization → Conga API Connections tab

Here you can create two types of connections:

After filling in the form, click Save & Generate Client Secret. The same one-time credentials popup appears. Conga recommends the Integration User method (Chapter 3) for most scenarios.
